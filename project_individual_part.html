Part1: Data Description

The players.csv contains the list of all unique players, including data about 7 features for 196 players. These columns include:

  • experience <chr>: Represents the player's level, including: Pro, Veteran, Regular, Amateur, and beginner.
  • subscribe <lgl>: subscribe state of game-related news (TRUE/FALSE).
  • hashedEmail <chr>: Players' email, encrypted by hash value.
  • played_hours <dbl>: Players' playtime in hours.
  • name <chr>: players' name.
  • gender <chr>: Gender, including: Male, Female, Non-binary, Prefer not to say, and Other.
  • Age <dbl>: Players' age.

Part2: Question

The broad question is: "which kinds of players are most likely to contribute a large amount of data so that we can target those players in our recruiting efforts?"

The specific question is "Can experience, subscribe, and age predict if a new volunteer can give a played_hours that is higher than the average in players.csv?".

To answer this, we need to simplify the players.csv dataset and select the needed variables in the prediction. In addition, before modelling the data, we should visualise it and see if these variables can reflect the result effectively. So that we could decide whether to use a linear regression model or a KNN model in the prediction.

#`hashedEmail`, `name`, `gender` cannot be averaged so they are skipped.
# For `experience`, although this column also exists in the form of \<chr>
# and cannot be averaged directly, we can add a new column `level` consisting of five 
# numbers from 1 to 5, which will be used to express the five levels in `experience`.

players_l <- players|>
    mutate(level = recode(experience,     # Write the corresponding level via recode()
                          "Beginner" = 1, 
                          "Amateur" = 2, 
                          "Regular" = 3,
                          "Veteran" = 4,
                          "Pro" = 5))

players_mean <- players_l|> 
    summarize(level_avg = mean(level),   # The corresponding mean value is obtained directly through summarize
              subscribe_avg = mean(subscribe), 
              played_hours_avg = mean(played_hours), 
              age_avg = mean(Age))

Part4: Methods and Plan

I think we should use the KNN prediction method.

Because the question is "predict if a new volunteer can give a played_hours that is higher than the average.", which is a binary problem where the prediction must be yes or no, and KNN model is better at this kind of classification work than linear regression.

However, there is a drawback to this method, when there are outliers in the data, the accuracy of the KNN model decreases significantly. Therefore, to choose the most correct model, we should use both methods for modelling and compare their RMSPE and accuracy to decide the better one.

The preprocessing of the data:

  • Creating a scatter plot and removing/adjusting any outliers.
  • Adjust ANY excessive difference between the values of the different variables.
  • Use cross-checking to determine the optimal number of "neighbours".
